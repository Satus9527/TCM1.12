【方案1: 在 Linux 服务器上部署（推荐）】

1. 准备 Linux 服务器
   - 云服务器（阿里云、腾讯云、AWS等）
   - 或虚拟机（VirtualBox、VMware）
   - 或 WSL2（Windows Subsystem for Linux）

2. 上传项目文件
   - 使用 Git 克隆: git clone YOUR_REPO_URL
   - 或使用 FTP 工具上传

3. 按照部署指南执行
   - 打开: 🚀_生产部署指南_无Docker版.md
   - 按五个阶段逐步执行
   - 使用提供的配置文件和脚本

4. 验证部署
   - 运行健康检查
   - 测试 API 端点

【方案2: 本地 Windows 开发】

1. 安装 Node.js v18
2. 安装依赖: npm install
3. 配置本地数据库
4. 启动: npm run dev
5. 注意: 仅用于开发测试，不是生产

================================================================
  🔧 配置文件使用
================================================================

ecosystem.config.js
  - 用于 PM2 进程管理
  - 复制到服务器: cp ecosystem.config.js /var/www/tcm-backend/
  - 启动: pm2 start ecosystem.config.js

nginx-tcm-platform.conf
  - Nginx 反向代理配置
  - 复制到: /etc/nginx/sites-available/tcm_platform
  - 启用: sudo ln -s ... /etc/nginx/sites-enabled/
  - 测试: sudo nginx -t
  - 重载: sudo systemctl reload nginx

deploy.sh
  - 自动化部署脚本
  - 首次部署后，用于后续更新
  - 使用: ./deploy.sh

================================================================
  📞 常见问题
================================================================

Q: 在 Windows 上能直接部署吗？
A: 不能。需要 Linux 服务器。

Q: 没有服务器怎么办？
A: 租用云服务器或使用虚拟机。

Q: 部署需要多长时间？
A: 首次 1-2 小时，后续更新 5-10 分钟。

Q: 部署脚本怎么用？
A: 在 Linux 服务器上执行: chmod +x deploy.sh && ./deploy.sh
